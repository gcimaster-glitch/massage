# HOGUSY 事業資料・システム統合報告書

**発行日:** 2026年3月16日
**発行者:** Manus AI プロフェッショナルチーム（年収2,500〜3,000万円クラス 5名構成）

---

## リード：結論

**事業資料とシステムの間に3つの重大な齟齬が確認されました。** 最優先で対応すべきは「収益分配モデルの不一致」です。資料では「セラピスト40% / オフィス25% / ホスト20% / 本部10% / 販促5%」と定義されているにもかかわらず、現システムは「セラピスト70% / 本部30%」というまったく異なる設定で稼働しています。この状態で本番稼働した場合、ステークホルダーへの報酬計算が事業計画と乖離し、信頼を損なう重大なリスクがあります。本報告書では、この問題を含む全齟齬の詳細と、段階的な解決計画を提示します。

## 2. 発見された齟齬の全体像

### 2.1. 齟齬マップ（重要度別）

| 齟齬の種類 | 資料の定義 | 現システムの状態 | 重要度 | ビジネスへの影響 |
| :--- | :--- | :--- | :--- | :--- |
| **収益分配モデル** | セラピスト40% / オフィス25% / ホスト20% / 本部10% / 販促5% | セラピスト70% / 本部30%（ハードコード） | **最高** | 報酬計算の誤り。本番稼働不可 |
| **役割名称の不統一** | 「セラピストオフィス」「拠点ホスト」 | `OFFICE`、「施設ホスト」が混在 | **高** | 顧客・パートナーの混乱 |
| **料金体系の未実装** | 月額：オフィス¥15,000〜¥25,000 / ホスト¥6,000 / セラピスト¥500 | DBに料金テーブルなし。コードに未定義 | **高** | 課金機能が存在しない |
| **施設タイプの定義** | CARE CUBE / ホテル / 自宅 / オフィス | `CARE_CUBE`, `HOTEL`, `PAUWAU`, `CHARGE`が混在 | **中** | 第三者への説明が困難 |
| **メールフッター** | HOGUSY | 「HOGUSY \| Powered by CHARGE」と表示 | **低** | ブランドの混乱 |

## 3. 齟齬の詳細分析

### 3.1. 収益分配モデルの不一致（最重要）

**FinTech専門家の評価:**

> 「現在のシステムが稼働したままでは、拠点ホストとセラピストオフィスへの報酬が一切支払われない。これは事業契約上の債務不履行に相当する可能性がある。本番稼働前に必ず修正すること。」

事業資料が定義する分配モデルは、施術代金¥10,000を例にとると以下のとおりです。

| 受取人 | 配分率 | 金額（¥10,000の場合） |
| :--- | :---: | ---: |
| セラピスト | 40% | ¥4,000 |
| セラピストオフィス | 25% | ¥2,500 |
| 拠点ホスト | 20% | ¥2,000 |
| HOGUSY本部（手数料） | 10% | ¥1,000 |
| 販促コスト | 5% | ¥500 |
| **合計** | **100%** | **¥10,000** |

現在のシステムでは、セラピストが70%、本部が30%を受け取る設定になっており、ホストとオフィスへの分配が**ゼロ**です。

### 3.2. 役割名称の不統一

**UI/UXデザイナーの評価:**

> 「同じ画面内に『施設ホスト』と『拠点ホスト』が混在している。これは第三者への説明資料を作成する際に、どちらが正式名称か分からなくなる原因となる。今すぐ統一すべき。」

現在、以下の表記が混在しています。

| 概念 | 資料上の正式名称 | システム内の現状表記 |
| :--- | :--- | :--- |
| 施術場所を提供する事業者 | **拠点ホスト** | 「施設ホスト」「拠点ホスト」「HOST」が混在 |
| セラピストを管理する法人 | **セラピストオフィス** | 「セラピスト事務所」「オフィス」「OFFICE」が混在 |

### 3.3. 料金体系の未実装

**BtoBコンサルタントの評価:**

> 「セラピストオフィスへの営業において、料金プランの提示は最重要の商談材料。これがシステムに存在しないということは、契約後の請求が手作業になることを意味する。スケールしない。」

資料が定義する月額料金プランは以下のとおりですが、現在のDBにはこれを管理するテーブルが存在しません。

| 対象 | プラン名 | 初期費用（加盟金） | 月額費用 |
| :--- | :--- | ---: | ---: |
| セラピストオフィス | バーチャル店舗 | ¥75,000 | ¥15,000 |
| セラピストオフィス | リアル店舗 | ¥150,000 | ¥25,000 |
| 拠点ホスト | CARE CUBE 1ブース | ¥250,000 | ¥11,000 |
| セラピスト | 個人利用 | ¥0 | ¥500 |

## 4. 統合方針と実装ロードマップ

### フェーズ1（即時対応：1〜2週間）

収益分配モデルの修正とロール名称の統一を最優先で実施します。これはシステムの本番稼働に直結する最重要タスクです。

- `src/types.ts` の `UserRole` 型を修正（`OFFICE` → `THERAPIST_OFFICE`）
- `src/revenue-engine-routes.ts` の分配ロジックを資料の数値に修正
- DBに `revenue_share_rules` テーブルを新設し、デフォルトルールを登録
- UI上の役割名称を全ファイルで統一

### フェーズ2（基盤強化：3〜4週間）

料金体系のDB管理化と管理画面の実装を行います。

- DBに `plans` テーブルを新設
- 管理者ダッシュボードに「料金プラン管理」画面を追加
- 管理者ダッシュボードに「収益分配ルール管理」画面を追加
- ユーザー登録フローに料金プラン選択を統合

### フェーズ3（品質保証：5〜6週間）

全ロールでのE2Eテストを実施し、本番稼働の準備を完了させます。

- 各ロールでの予約〜決済〜報酬分配の一連フローのテスト
- 異なる施設タイプ（CARE CUBE / ホテル）での分配ルール適用の検証
- 外部公開前のデモデータ削除と`/dev-login.html`の削除
